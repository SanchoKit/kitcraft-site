KitCraft

Personal Minecraft server project running in Docker with BlueMap. Hosted on my own machine for now, easy to move to a Raspberry Pi 5 later.

How to join

Edition: Java (Paper 1.21.x)

Server address: later-classroom.gl.joinmc.link

This goes through Playit.gg, which tunnels traffic to my local 25565.

Live map

BlueMap is exposed through a second tunnel. Open the live map here:

http://minimum-mathematics.gl.at.ply.gg:26308

Under the hood this is just forwarding to localhost:8100 on my machine.

World details

  • Seed: 1331376639841402888
  • World size: 1000 × 1000 blocks (worldborder around spawn)
  • Style: vanilla-ish survival, mostly for experimenting and building

Docker / Compose setup

The server runs as a single container using itzg/minecraft-server. All data lives in a bind-mounted folder so I can move it to another machine without reinstalling anything.

docker-compose.yml

services:
  mc-server:
    image: itzg/minecraft-server
    container_name: mc-server
    ports:
      - "25565:25565"   # Minecraft
      - "8100:8100"     # BlueMap
    environment:
      EULA: "TRUE"
      TYPE: "PAPER"
      VERSION: "1.21.1"
    volumes:
      - "./minecraft-data:/data"
    restart: unless-stopped

Data directory: ./minecraft-data

  • worlds
  • plugins (including BlueMap)
  • configs / logs

Because everything is in this folder, I can copy minecraft-data and docker-compose.yml to a Raspberry Pi or cloud VM and run docker compose up -d to get the same server.

Why I'm doing it this way

This project is mainly for learning:

  • running game servers in containers (instead of directly on the host),
  • keeping world data outside the container so it’s portable,
  • exposing services safely over the internet using tunnels instead of raw port forwarding,
  • and eventually moving the same setup to a Raspberry Pi or a cloud VM.

Resource usage (rough)

With Paper + BlueMap, the container sits around ~1.5–2 GB RAM after startup on my machine. CPU spikes during world generation and map rendering, then mostly idles.
